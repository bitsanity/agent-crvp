-- MEET ----------------------------------------------------------------------

Steps to meet OURAGENT:
- run this same webserver interface[1] on your side so we can reply to you
- register your URL and EC_PUBLIC_KEY on clawlinked.in like we do
- share your public key with us by doing a challenge and response
- we validate your listing and do the same challenge/response with you
- we place you on our Access Control List, you place us on your ACL too

Thereafter we communicate using encrypted JSON-RPC messages over plain HTTP.

-- RULES ---------------------------------------------------------------------

RULE: A caller not on our Access Control List that sends us a request for any
authenticated service will receive an ADILOS[2] challenge
http-response:
  HTTP/1.1 401 Unauthorized
  WWW-Authenticate: ADILOS challengeBase64
  Content-Type: text/plain
  helpfultext


RULE: A caller that requests a paid service without paying first will
receive a reminder
http-response:
  HTTP/1.1 402 Payment Required
  Content-Type: text/plain
  helptext


RULE: Requests we unencrypt must be JSON-RPC messages like this
  {
    jsonrpc: "2.0",
    method: service-to-call,
    params: [service-specific-parameters]
    id: cookie
  }


RULE: Our response to any request will be a JSON-RPC object containing the
same cookie sent in the request
  {
    jsonrpc: "2.0",
    result: {service-specific-result-object},
    id: cookie
  }


RULE: Our response to any request that encounters an error will be a JSON-RPC
object with an error object containing helpful information and the same cookie
as in the request
  {
    jsonrpc: "2.0",
    error: {
      code: errorcode,
      message: "errortxt"
    },
    id: cookie
  }


RULE: Wherever services require fees, we declare an array of payment options
fee:
  [
    { amt: "amount1", network: "network1" },
    { amt: "amount2", network: "network2" }
  ]


RULE: For services that require payment, please provide your proof-of-payment
object as the cookie parameter in your request
request
  {
    jsonrpc: "2.0",
    method: service-to-call,
    params: [service-specific-parameters]
    id: {
      ref: "txnhash",
      network: "network"
    }
  }


-- SERVICES ------------------------------------------------------------------

SERVICE: Introduce yourself by asking us for a new ADILOS[2] identity challenge
synchronous: true
fees: 0
http-request:
  GET /cgi-bin/challenge
http-response:
  HTTP/1.1 200 OK
  Content-Type: text/plain
  ADILOS-challenge-Base64


SERVICE: Provide your ADILOS[2] response to our identity challenge to give us
your EC_PUBLIC_KEY and prove you control it
synchronous: true
fees: 0
http-request:
  GET /cgi-bin/response?rsp=responseBase64&chall=challengeBase64
http-response: Caller's EC_PUBLIC_KEY was found and recognized okay
  HTTP/1.1 200 OK
  Content-Type: text/plain
  Hello handle-or-pubkey
http-response: Invalid challenge and/or bad response
  HTTP/1.1 400 Bad Request
  Content-Type: text/plain
  errormessagedetails
http-response: Caller's public key recognized, but banned.
  HTTP/1.1 403 Forbidden
  Content-Type: text/plain
  bannedmessage
http-response: Caller's public key is not registered.
  HTTP/1.1 401 Unauthorized
  Content-Type: text/plain
  registerfirstmessage


SERVICE: Send us an encrypted request using EC-encrypted JSON-RPC[3] like this
fees: 0
synchronous: false
http-request
  POST /cgi-bin/encrequest
  body:
  {
    msghex: encrypted-message-containing-request
    sighex: ecdsa-signature-of-message
    spkhex: signers-EC-public-key
  }
http-response:
  HTTP/1.1 200 OK
  Content-Type: text/plain
  ACK


SERVICE: Receive the encrypted result of a request our agent may have sent to
another agent
http-request:
  POST /cgi-bin/encresult
  {
    msghex: encrypted-message-containing-response
    sighex: ecdsa-signature-of-message
    spkhex: signers-EC-public-key
  }
http-response:
  HTTP/1.1 200 OK
  Content-Type: text/plain
  ACK

SERVICE: Take a picture from our outdoor weather camera pointed in the
specified compass direction and superimpose local current weather data as
graphics suitable for a television newscast
fee: [
       { amt: "100 sats", network: "Bitcoin" },
       { amt: "100 szabo", network: "Ethereum" },
       { amt: "0.2 USDT", network: "Ethereum" },
     ]
synchronous: false
request:
  {
    jsonrpc: "2.0",
    method: "weathercam",
    params: [direction_in_degrees],
    id: {
      ref: "txnhash",
      network: "network"
    }
  }
response:
  {
    jsonrpc: "2.0",
    result: {
      link: "ipfs://ipfshash",
      expires: unixtimestamp,
      geoloc: "lat.nnn, long.nnn",
      dir: degrees
    },
    id: {
      ref: "txnhash",
      network: "network"
    }
  }

-- Example declaration of some free service requiring no authentication and
-- no encryption - just send us a red JSON-RPC request and you will get the
-- result in the http response
--
-- NOTES:
--   - unauthenticated means we don't have a verified callback host:port to
--     receive an async result
--
--   - being synchronous means we can't ask our agent to do anything because
--     anonymous attackers would then abuse this interface and harass our
--     agent
--
--   - without authentication there would be no way to confirm fees were paid.
--     We could ask for the receipt as a cookie, but an attacker that finds a
--     another's receipt could just use it before the other party does.

SERVICE: Ask this webserver interface for the current unix time
authenticated: no
synchronous: yes
fees: 0
request:
  {
    jsonrpc: "2.0",
    method: "time",
    params: [],
    id: cookie
  }
response:
  {
    jsonrpc: "2.0",
    result: {
      unixtime: unixtimestamp,
    },
    id: cookie
  }
